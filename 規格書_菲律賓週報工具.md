## 2. 資料來源與蒐集策略

工具背後的「內容整理人」是 Claude（也就是我）。每週一 9:00 我會自動執行以下流程：

### 2.1 第一輪掃描：找出本週候選話題

從以下來源蒐集過去 7 天的菲律賓話題：

1. **Google Trends Philippines**
   - 即時熱門：`trends.google.com/trends/trendingsearches/realtime?geo=PH`
   - 過去 7 天榜：`trends.google.com/trends/explore?geo=PH`
   - 取前 20 名搜尋詞做初步篩選

2. **Reddit 菲律賓相關 subreddit**（按優先序）
   - `r/Philippines`：本週 Top 帖
   - `r/OffMyChestPH`：高互動帖
   - `r/adultingph`：本週 Top 帖
   - `r/phinvest`、`r/phcareers`：列為參考

3. **菲律賓主流媒體娛樂版**
   - PEP.ph（八卦娛樂專業媒體）
   - Spot.ph（生活風格與流行文化）
   - Rappler 的 Life & Style 版
   - GMA News Lifestyle 版

4. **遊戲與電競**
   - One Esports Philippines（`philippines.oneesports.gg`）
   - Mobile Legends Bang Bang 官方賽事公告
   - YouTube 菲律賓遊戲類熱門

5. **TikTok / X / Facebook 公開內容**
   - 透過 Web 搜尋抓「Philippines viral TikTok this week」「Philippines trending Twitter」等語句
   - X 的菲律賓地區趨勢（網頁版可看，不需 X Pro）

### 2.2 第二輪篩選：判斷哪些值得進卡片牆

從第一輪蒐集到的清單中，依以下準則篩出 8–12 張：

- **是否符合「社群 > 娛樂 > 遊戲」的領域優先序？**
  目標分配：社群 4–5 張、娛樂 3–4 張、遊戲 2–3 張（可微調，視當週熱度）
- **是否是「梗」或具有「梗的潛力」？**（這是核心判準）
  - 有特定 hashtag、特定動作／句子／畫面被反覆模仿 → 強烈是
  - 純政治／純財經事件 → 過濾掉（除非衍生出梗）
  - 名人緋聞、節目換主持、影視作品上映等若有衍生梗 → 收
- **是否有足夠的「可講性」？**
  - 行銷團隊能拿來改編、引用、做內容 → 收
  - 純內幕、太地區化、難以解釋給台灣讀者 → 過濾掉
- **24 小時內熱度暴漲的優先**：給【爆】徽章
- **本質是 Meme（非事件衍生）**：給【純梗】徽章

### 2.3 第三輪生成：寫卡片內容

每張卡片寫：

- **標題**（中文，含關鍵字括號）— 例：`#KimChiuWedding 婚禮現場改編梗（藝人 Kim Chiu）`
- **一行摘要**（卡片預設顯示）— 30–50 字，講清楚是什麼
- **完整內容**（點擊展開後顯示）—
  - 「這是什麼」：背景故事 2–3 句
  - 「為什麼紅」：傳播路徑、平台
  - 「具體熱度」：能查到的數字（觀看數、分享數、起源時間）
  - 「行銷可用性」：給一句建議該怎麼借勢
  - 「原文連結／關鍵字」：1–3 個可直接搜尋的字串
- **領域標籤**：社群 / 娛樂 / 遊戲（單選）
- **徽章**：【爆】、【純梗】（可同時擁有 0–2 個）
- **熱度分數**：1–100，用於排序

### 4.2 卡片狀態

- **收合狀態**（預設）：
  - 上方：徽章＋領域標籤
  - 中央：標題（中文＋原文括號）
  - 下方：30–50 字摘要 + 「展開」按鈕
- **展開狀態**：
  - 顯示完整 `detail` 物件的五個欄位
  - 提供「複製全文」按鈕（複製為純文字方便貼到內容企劃文件）
  - 提供「收合」按鈕

### 4.3 篩選與切換

- **領域篩選**：點「社群／娛樂／遊戲」只顯示該類；點「全部」回到不過濾
- **徽章篩選**：可獨立勾選「僅看 爆」「僅看 純梗」
- **週次切換**：右上下拉選單列出最近 4 週；選了之後卡片牆即時更新
- **手動重整**：右上「🔄 重整」按鈕點下去會跳出對話框「將觸發 Claude 重新整理本週內容，可能需要 2–3 分鐘。確定嗎？」（實作上會用 `sendPrompt` 把指令送到對話）

### 4.4 視覺風格

- 配色：以柔和的灰白為底，菲律賓國旗藍紅為點綴
- 字型：思源黑體 / 微軟正黑體
- 響應式：桌面螢幕 1024px 以上單欄寬卡片；手機尺寸亦可閱讀
- 暗色模式：第一版不做，未來可加

## 5. 自動化排程

### 5.1 觸發時間

- **每週一 09:00（Asia/Taipei，UTC+8）**
- 對應 cron 表達式：`0 9 * * 1`
- 您在新加坡時這個時間是當地時間 09:00（新加坡與台北同時區）

### 5.2 任務執行流程

1. Claude（我）收到排程觸發
2. 執行第 2 章所述的三輪蒐集／篩選／生成
3. 把本週 8–12 張卡片資料寫進 `菲律賓話題週報.html` 內嵌的 JSON
4. 若舊資料存在，把上週與更早的資料一起保留在同一個 HTML 內（最多保留 8 週，超過自動刪最舊）
5. 完成後在 Cowork 對話留下訊息：「本週菲律賓話題週報已更新，共 X 張卡片，最熱：[第一張標題]」

### 5.3 失敗處理

- 若 WebSearch／WebFetch 任何一步失敗：先重試 1 次
- 仍失敗：在 HTML 內顯示「本週資料未能完整更新（[失敗原因]），顯示的是上週內容」橫幅，且不覆蓋舊資料
- 失敗訊息會同步發到 Cowork 對話讓您知道

### 5.4 手動觸發方式

您隨時可以在 Cowork 對話對 Claude 說「請更新本週菲律賓話題週報」，會立刻執行同樣流程。

### 7.2 為什麼選這種架構（非工程師可跳過）

- **單檔自給自足**：不需要伺服器、不需要 API、不需要 CORS 處理、不需要使用者開 VPN，最低門檻
- **由 Claude 端整理資料、產出 HTML**：規避了瀏覽器無法跨域抓取 RSS／API 的限制（這正是您附件方案實際會遇到的問題）
- **資料內嵌而非外鏈**：未來即使所有來源網站全部下線，您手上的 HTML 還是能看到過去的內容

### 7.3 不做的事

第一版明確「不做」：

- 不做帳號登入／個人化
- 不做使用者自行加新話題的功能
- 不做評論／收藏／分享功能
- 不做行動 App
- 不做後端伺服器
- 不串接任何付費 API
- 不做 KOL 追蹤頁面（保留為未來功能）
