## 2. 資料來源與蒐集策略

工具背後的「內容整理人」是 Claude。每週一 09:00 我會自動執行以下流程：

### 2.1 第一輪掃描：找出本週候選話題

從以下來源蒐集過去 7 天的菲律賓話題：

1. **Google Trends Philippines**
   - 即時熱門：`trends.google.com/trends/trendingsearches/realtime?geo=PH`
   - 過去 7 天榜：`trends.google.com/trends/explore?geo=PH`
   - 取前 20 名搜尋詞做初步篩選

2. **Reddit 菲律賓相關 subreddit**（按優先序）
   - `r/Philippines`：本週 Top 帖
   - `r/OffMyChestPH`：高互動帖
   - `r/adultingph`：本週 Top 帖
   - `r/phinvest`、`r/phcareers`：列為參考

3. **菲律賓主流媒體娛樂版**
   - PEP.ph（八卦娛樂專業媒體）
   - Spot.ph（生活風格與流行文化）
   - Rappler 的 Life & Style 版
   - GMA News Lifestyle 版

4. **遊戲與電競**
   - One Esports Philippines（`philippines.oneesports.gg`）
   - Mobile Legends Bang Bang 官方賽事公告
   - YouTube 菲律賓遊戲類熱門

5. **TikTok / X / Facebook 公開內容**
   - 透過 Web 搜尋抓「Philippines viral TikTok this week」「Philippines trending Twitter」等語句
   - X 的菲律賓地區趨勢（網頁版可看，不需 X Pro）

### 2.2 第二輪篩選：判斷哪些值得進卡片牆

- **領域配比**：社群 4–5 張、娛樂 3–4 張、遊戲 2–3 張（可微調）
- **是否是「梗」或具有「梗的潛力」**
- **可講性**：行銷團隊能拿來改編、引用、做內容
- **政治／災難敏感內容**：預設過濾（除非衍生出明確的非政治性 Meme）
- **24 小時內熱度暴漲的優先**：給【爆】徽章
- **本質是 Meme（非事件衍生）**：給【純梗】徽章
- **平台來源追溯**（v1.1 新增）：判斷話題起源平台與延燒平台，並記錄各平台首次出現的日期

### 2.3 第三輪生成：寫卡片內容

每張卡片寫：

- **標題**（中文＋關鍵字括號）
- **一行摘要**（30–50 字）
- **完整內容**：「這是什麼」「為什麼紅」「具體熱度」「行銷可用性」「來源平台」「搜尋關鍵字」六大欄位
- **領域標籤**：社群／娛樂／遊戲（單選）
- **徽章**：【爆】、【純梗】（可同時擁有 0–2 個）
- **來源平台**（v1.1 新增）：陣列形式記錄各平台 + 角色（起源／延燒／熱議／報導）+ 首見日期
- **熱度分數**：1–100，用於排序

### 4.2 卡片狀態

**收合（預設）**：徽章列＋標題＋一行摘要＋來源徽章列（最多顯示 6 個，超過顯示「+N more」）＋「展開」按鈕

**展開**：完整顯示六個 detail 欄位 + 「複製全文」「收合」按鈕。展開狀態的「來源平台」會帶上日期（如「TikTok 起源 5/15、FB 延燒 5/16⋯」），讓使用者看到話題如何在平台間擴散。

### 4.4 篩選與切換

- **領域篩選**：點「社群／娛樂／遊戲」只顯示該類；點「全部」回到不過濾
- **徽章篩選**：可獨立勾選「僅看 爆」「僅看 純梗」（可同時勾）
- **週次切換**：右上下拉選單列出最近 8 週；選了之後卡片牆即時更新，副標顯示週次區間
- **歷史週檢視標示**（v1.1 新增）：副標題會出現橘色「（檢視歷史週）」字樣
- **重整按鈕禁用**（v1.1 新增）：檢視歷史週時，「🔄 重整」按鈕自動變灰，無法點擊（因為過往週不應重生）
- **手動重整**：因為這是單檔 HTML，無法直接觸發 Claude。點「🔄 重整」會彈出小視窗：「請至 Cowork 對話視窗對 Claude 說『請更新本週菲律賓話題週報』。點此複製指令到剪貼簿。」

### 4.5 視覺風格

- 配色：以柔和的灰白為底，菲律賓國旗藍紅為點綴
- 字型：思源黑體 / 微軟正黑體（Windows、Mac 通用）

## 5. 自動化排程

### 5.1 觸發時間

- **每週一 09:00（Asia/Taipei，UTC+8）**
- 對應 cron 表達式：`0 9 * * 1`
- 您在新加坡時這個時間是當地時間 09:00（同時區）
- 若 Cowork App 在排程時間未開啟，會在下次開啟時補跑（會通知您）

### 5.2 完成通知格式（v1.1 補強）

排程跑完後在 Cowork 對話留下訊息：

```
✅ 本週菲律賓話題週報已更新

📅 期間：2026-05-18 ~ 2026-05-24
📊 共 10 張卡片（社群 4 / 娛樂 4 / 遊戲 2）
🔥 最熱：#BibitawNaDanceChallenge 引爆全民跟拍

【標記】2 張掛「爆」、3 張掛「純梗」
【來源涵蓋】TikTok 7 張、FB 6 張、X 5 張、Rappler 3 張、Reddit 2 張

點此打開：菲律賓話題週報.html
```

### 5.3 失敗處理

- 若 WebSearch／WebFetch 任何一步失敗：先重試 1 次
- 仍失敗：在 HTML 內顯示「本週資料未能完整更新（[失敗原因]），顯示的是上週內容」橘色橫幅，且不覆蓋舊資料
- 失敗訊息會同步發到 Cowork 對話讓您知道

### 5.4 手動觸發方式

您隨時可以在 Cowork 對話對 Claude 說「請更新本週菲律賓話題週報」，會立刻執行同樣流程。

```html
    /* 渲染與互動 */
  </script>

```

### 7.2 為什麼選這種架構

- **單檔自給自足**：不需要伺服器、不需要 API、不需要 CORS 處理、不需要使用者開 VPN
- **由 Claude 端整理資料、產出 HTML**：規避瀏覽器無法跨域抓取 RSS／API 的限制
- **資料內嵌而非外鏈**：未來即使所有來源網站全部下線，您手上的 HTML 還是能看到過去內容
- **「複製到任何電腦都能用」**：把 HTML 寄給同事看，他們不需要 VPN、不需要安裝任何東西

### 7.3 不做的事（v1）

- 不做帳號登入／個人化
- 不做使用者自行加新話題的功能
- 不做評論／收藏／分享功能
- 不做行動 App
- 不做後端伺服器
- 不串接任何付費 API
- 不做 KOL 追蹤頁面（保留為未來功能）

## 8. 上線後的維運注意事項

### 8.1 您需要做的事

- 每週一打開 HTML 看一下，確認排程有跑、內容有更新
- 若連續兩週發現「內容品質不對」或「方向偏掉」，告訴我，我會校正篩選邏輯

### 8.2 我會做的事

- 每週一 9:00 自動執行
- 失敗時主動通知您
- 您回饋後調整篩選邏輯

### 8.3 風險與限制

- **WebSearch 有時會失敗或被限速**：發生時回到上週內容並通知您
- **菲律賓本地 Meme 有語境門檻**：前 2–3 週可能會偶爾收進不太精準的項目，需迭代
- **未經本人授權的明星梗**：盡量避免帶入造謠／侵犯隱私的內容
- **政治敏感梗**：v1 預設過濾掉純政治內容，僅在政治話題衍生出明確非政治 Meme 時才收
